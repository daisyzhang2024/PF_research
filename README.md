## Quick Start

### 1. Install dependencies

```bash
pip install requests beautifulsoup4 camelot-py pdfplumber pandas
# For scanned PDFs (pre-2018 era):
apt install ocrmypdf   # or: brew install ocrmypdf
```

### 2. Run the scraper

```bash
cd scripts/

# Full run: discover + download + extract
python idor_scraper.py

# Download only (no extraction)
python idor_scraper.py --download-only

# Extract from already-downloaded PDFs
python idor_scraper.py --extract-only

# Process a specific year
python idor_scraper.py --year 2017

# Force re-download and re-extract everything
python idor_scraper.py --force
```

### 3. Handle scanned PDFs

PDFs from approximately 2015–2017 may be scanned images. After the scraper
flags them as `"extraction_status": "no_rows"` in the manifest:

```bash
cd output/raw_pdfs/

# OCR a single file
ocrmypdf input.pdf input_ocr.pdf --deskew --clean

# Batch OCR all PDFs flagged as no_rows
# (check manifest.json for the list)

# Re-extract after OCR
cd ../../scripts/
python idor_scraper.py --extract-only --force
```

## Output Schema — `cook_county_chmr_series.csv`

| Column | Description |
|--------|-------------|
| `source_file` | PDF filename the record was extracted from |
| `report_year` | Calendar year inferred from PDF filename/label |
| `period` | Reporting month (e.g., "Jan 2017") — may be empty if not parseable |
| `government` | Always "COOK COUNTY" |
| `tax_code` | Always "CHMR" |
| `amount` | Dollar disbursement amount |
| `raw_row` | Raw pipe-delimited cell contents (for audit/debugging) |
| `extracted_at` | UTC timestamp of extraction |

**Version control:** The `manifest.json` file tracks:
- SHA-256 hash (first 12 chars) of each downloaded PDF
- Extraction status per file
- Number of CHMR records found per file
- Download and extraction timestamps

Re-running the scraper is idempotent unless `--force` is passed.

## Known Issues & Limitations

### Extraction reliability
- Camelot performs well on digitally-typed IDOR PDFs (2018+)
- Pre-2018 scanned PDFs require OCR preprocessing
- Multi-column layouts sometimes cause camelot to misalign cells — the `raw_row`
  column allows manual correction
- The `period` field may be empty if the month/year cannot be parsed from the row
  itself; in those cases, infer from the PDF filename and report_year

### Data gaps
- The archive index page uses a CMS asset manager; if the dynamic asset list
  doesn't fully render with requests, consider using Selenium or Playwright:
  ```bash
  pip install playwright && playwright install chromium
  ```
  Then replace the `discover_pdf_urls()` function with a headless browser call.

### Structural breaks in the series
See `docs/data_source_inventory.md` Part 2 for full documentation of:
1. Portal migration discontinuity (~2021/2022)
2. PA 103-0983 policy break (January 2025)
3. COVID-19 disruption window (March–September 2020)

## Data Source Priority Summary

| Source | Priority | Use Case |
|--------|----------|----------|
| IDOR Live Portal (2022+) | P0 | Primary monthly CHMR data, recent |
| IDOR Archive (2006–2021) | P0 | Primary monthly CHMR data, historical |
| Cook County OCFO Methodology Docs | P1 | Understand official forecast assumptions |
| Cook County ACFRs | P1 | Annual cross-validation |
| IRFC Meeting Packets (Legistar) | P1 | Quarterly variance data |
| SafeGraph Patterns (via Dewey) | P0 | Foot traffic covariate — highest priority novel data |
| Placer.ai | P2 | Foot traffic backup / alternative methodology |
| IDOR Annual Reports | P2 | Gap-fill, cross-validation only |
